# Part 2: Generate the ODD protocol for your model

Once your odder plugin is built, ODD generation follows a 4-step manual process. We recommend one step per agent session. After each step, artifacts are generated by the agent and should be reviewed by the human.

Install the plugin:
```sh
claude --plugin ../path/to/the/generated/plugin
```
Run the sessions in order:
```sh
1. /odder:interview
2. /odder:plan
3. /odder:draft
4. /odder:check
```

## Session 1: Interview/Research (`/interview`)

The agent reviews all available source materials — documentation, code, papers — and conducts a structured interview with you about anything missing or ambiguous. It researches your model's documentation, code, and anything else you give it access to, then asks about missing information required for ODD generation.

The interview strategy adapts based on input quality (see the Adaptive Interview Strategy table above).

**Output:** `odder/research/findings.md` — structured findings organized by ODD element, with confidence annotations on every claim.

**Your job:** Review the findings. Correct misunderstandings, fill gaps, confirm inferences.

## Session 2: Plan (`/plan`)

The agent takes the reviewed research findings and plans the generation of the ODD. Because generation is complicated and might involve multiple agents/sub-agents, this phase creates a standalone generation plan — essentially a detailed "mega-prompt" that contains everything needed to draft the ODD without access to the original sources.

**Output:** `odder/plan/odd-generation-plan.md` — self-contained instructions for the drafting agent.

**Your job:** Review the plan. Check that the intended structure, level of detail, and emphasis match what you want.

## Session 3: Draft (`/draft`)

A fresh agent (with no memory of the research phase) executes the generation plan. This is the actual generation of the ODD. It produces the full ODD document following the ODD+2 protocol structure, plus a traceability matrix mapping every claim to its source.

**Output:** `odder/draft/odd.md` + `odder/draft/traceability-matrix.md`

**Your job:** Read the ODD. Check that it accurately represents your model and that the traceability matrix citations are correct.

## Session 4: Verify (`/check`)

An independent verification agent checks the ODD against the original source materials. It does NOT see the generation plan — only the ODD and the original docs/code. It runs structural completeness checks, code alignment checks, semantic consistency checks between sections, a confidence audit, and the reimplementability test.

**Output:** `odder/checked/verification-report.md` — scored rubric with specific findings.

**Your job:** Address any issues flagged in the verification report. Iterate as needed.
